Xbox 360 Hacks Guide

Unlock the Xbox 360 to run unsigned code, homebrew, emulators, and backups via a glitch- or NAND-based exploit.

What this is for

Xbox 360 modding exploits the boot chain to run a modified kernel that allows unsigned code - enabling homebrew, the Aurora/FreeStyle dashboards, emulators, and games from storage. The method you can use depends entirely on your motherboard revision and CB (bootloader) version, since later kernels and CBs are patched.

⚙️ Methods, versions & compatibility — read this first

MethodWorks onNotes
JTAG
Exploit		Phat consoles on dashboard 7371 or lower with an unpatched CBSoftware/NAND exploit, instant boot, no glitch hardware. Impossible on consoles updated past 7371 or with patched CBs.
RGH 1.2
Exploit		Phats and slims (any dashboard); needs a glitch chipRefined reset-glitch hack; the more reliable fallback on stubborn Jasper/Tonka boards.
RGH 2 / S-RGH
Exploit		Phats and slims; EXT_CLK variant on Xenon/ZephyrDifferent solder points; faster boots on some boards. Still uses a glitch chip.
RGH 3
Exploit		Any console on dashboard newer than 2.0.14699No glitch chip - typically just two wires. Corona V3+ needs a Post Fix Adapter; can be slow on Jasper phats.
R-JTAG
Exploit		Phats with a dashboard higher than 7371Uses a glitch chip + RGH to re-enable the otherwise-patched JTAG exploit.

What to check first: Identify motherboard revision and CB/dashboard version FIRST. JTAG needs a phat at dashboard 7371 or lower with an unpatched CB. RGH works on any dashboard; Trinity/Corona slims are essentially 100% RGH-able. Corona 4GB consoles need a 4GB-capable flasher (PicoFlasher/xFlasher), and Corona V3+ needs a Post Fix Adapter for RGH3.

⚠️ Before you start: Console modding carries real risk — an interrupted install or the wrong method/file for your exact model can leave hardware unbootable. Identify your model first, back up anything important, and proceed at your own risk.

What you'll need

  • An Xbox 360 (identify phat vs slim and exact motherboard: Xenon/Zephyr/Falcon/Opus/Jasper/Trinity/Corona)
  • A NAND programmer - xFlasher 360, NAND-X, JR-Programmer, or a Raspberry Pi Pico (PicoFlasher)
  • J-Runner with Extras on a PC to build and write the image
  • Soldering iron, fine wire, and (for glitch methods) a glitch chip; Corona V3+ also a Post Fix Adapter
  • A USB drive or HDD for XeLL output and dashboard/homebrew files
  • Your console's CPU key (extracted from the NAND dump)

How it works — the process

This walkthrough names the tools at each stage so you know what each one is and what it's for. Always use the latest release of each named tool.

  1. Identify the console
    Determine motherboard revision and CB/dashboard version to choose between JTAG, RGH 1.2/2/3, or R-JTAG.
  2. Dump the NAND
    Wire a NAND programmer and read the stock NAND in J-Runner to extract the CPU key.
  3. Install the glitch/exploit
    Solder the glitch chip and timing wires (RGH1.2/2), the two RGH3 wires, or the JTAG wiring - adding a Post Fix Adapter on Corona V3+.
  4. Build and write the hacked NAND
    In J-Runner, build the hacked image with your CPU key and flash it back.
  5. Boot XeLL and finalize
    Confirm the console glitches into XeLL, capture the CPU key, then write the final image and boot the hacked kernel.
  6. Set up dashboard and content
    Install Aurora/FreeStyle Dash and homebrew, configure storage, and add games or emulators.

Which option should I pick?

Phat on dashboard 7371 or lower with unpatched CB: JTAG is simplest (no glitch chip). Otherwise RGH: RGH3 (glitch-chip-free, just wires) is preferred on most boards, with RGH1.2/2 the reliable fallback on stubborn Jasper phats.
