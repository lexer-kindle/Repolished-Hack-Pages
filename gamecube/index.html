GameCube Hacks Guide

Run the all-in-one Swiss utility via a chosen entry point to load homebrew and backups from SD media.

What this is for

GameCube homebrew centers on Swiss, an all-in-one open-source utility that loads homebrew, plays backups, and patches discs from SD cards or ODE devices. Because there's no online OS to exploit, you get Swiss running through an entry point - a save-game exploit, an SD adapter, a boot disc, or a PicoBoot modchip - then browse and run files from FAT32 SD media.

What to check first: The GameCube has no system firmware; the key thing to check is your MODEL and ports. DOL-001 has the high-speed Serial Port 2 (for SD2SP2) and digital AV out; DOL-101 lacks digital video out, and a few DOL-001 rev 1.1 boards have no physical SP2 port. Format all SD media as FAT32 with Swiss and your files before starting.

⚠️ Before you start: Console modding carries real risk — an interrupted install or the wrong method/file for your exact model can leave hardware unbootable. Identify your model first, back up anything important, and proceed at your own risk.

What you'll need

  • GameCube console (DOL-001 or DOL-101) + controller
  • Latest Swiss release (swiss.dol) on FAT32 SD/microSD
  • A chosen entry point: SD2SP2, SD Gecko/SD Media Launcher, save exploit, PicoBoot, or ODE
  • A memory card (for the save-exploit method)
  • Soldering tools (only for PicoBoot or internal SD2SP2 installs)
  • A PC with an SD card reader

How it works — the process

This walkthrough names the tools at each stage so you know what each one is and what it's for. Always use the latest release of each named tool.

  1. Identify your model & ports
    Check whether you have a DOL-001 (Serial Port 2 present) or DOL-101, and confirm an SP2 port exists if using SD2SP2.
  2. Prepare SD media
    Format an SD card to FAT32 and copy the latest Swiss build (swiss.dol) plus your homebrew and ISOs.
  3. Choose an entry point
    Pick how Swiss first launches: a save exploit, the SD Media Launcher boot disc, an ODE, or a PicoBoot modchip.
  4. Install hardware (if needed)
    Fit an SD2SP2 in Serial Port 2; or flash a Pi Pico and solder ~5 wires for PicoBoot; or replace the drive for an ODE.
  5. Boot Swiss
    Launch Swiss through your entry point, then switch the device to your SD card and browse your files.
  6. Run homebrew & backups
    From Swiss, launch .dol/.elf homebrew or boot game ISOs with the right patch/boot settings.

Which option should I pick?

Cheapest no-solder route: a save exploit plus an SD Gecko/SD2SP2 for storage. Cleanest permanent route: PicoBoot, which auto-loads Swiss at power-on but requires soldering.
